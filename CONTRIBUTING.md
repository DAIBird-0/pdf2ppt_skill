# Contributing

Thanks for improving `pdf2ppt_skill`.

Before opening a pull request:

1. Do not commit private papers, generated decks, local work folders, tokens, API keys, or absolute machine paths.
2. Run the validation commands in `README.md`.
3. If you add a demo built from third-party paper content, document the source in `THIRD_PARTY_NOTICES.md`.
4. Keep `paper-group-meeting-ppt/SKILL.md` concise enough for agent context and move detailed rules into `references/`.

Pull requests should explain the user-visible workflow change and include screenshots or preview images when visual behavior changes.
